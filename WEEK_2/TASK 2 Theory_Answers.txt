Q12. If you run SELECT * FROM customers WHERE YEAR(join_date) = 2024;, would the index on join_date be used? Explain why or why not, and rewrite the query to be index-friendly.

Answer:

No, the index would generally not be used efficiently because the YEAR() function is applied to the join_date column. The database has to calculate the year for every row before filtering the results.

A better approach is to use a date range:

select *
from customers
where join_date >= '2024-01-01'
and join_date < '2025-01-01';

This type of query is more efficient because the database can use the index directly.

Q22. Explain the difference between LEFT JOIN and RIGHT JOIN with an example from this schema. When would you use a FULL OUTER JOIN?

Answer:

A LEFT JOIN returns all records from the left table and matching records from the right table. If there is no match, NULL values are returned for the right table columns.

A RIGHT JOIN returns all records from the right table and matching records from the left table. If there is no match, NULL values are returned for the left table columns.

For example, if we use a LEFT JOIN between customers and orders, all customers will be displayed even if they have not placed any orders.

A FULL OUTER JOIN is used when we want all records from both tables, including unmatched records from either side.

Q26. Explain each letter of ACID. Give a real-world example.

Answer:

ACID properties help maintain reliability and correctness in database transactions.

A – Atomicity

Atomicity means a transaction is completed fully or not completed at all. If any step fails, the entire transaction is rolled back.

C – Consistency

Consistency ensures that the database remains valid before and after a transaction. All rules and constraints must be satisfied.

I – Isolation

Isolation means multiple transactions can run at the same time without affecting each other.

D – Durability

Durability means once a transaction is committed, the changes are permanently stored even if the system crashes.

Example:

Consider a bank transfer of ₹1000 from Account A to Account B.

Atomicity ensures both debit and credit happen together.
Consistency ensures account balances remain correct.
Isolation prevents interference from other transactions.
Durability ensures the transfer remains saved after it is committed.
Q27. Write a SQL transaction that performs the given operations atomically.

Answer:

The transaction inserts a new order, adds order items, updates product stock, and commits all changes together.

If any step fails, the transaction should be rolled back so that no partial changes remain in the database.

start transaction;

insert into orders
values
(1011, 102, curdate(), 'Pending', 1598.00);

insert into order_items
values
(5016, 1011, 206, 1, 1299.00, 0);

insert into order_items
values
(5017, 1011, 208, 1, 299.00, 0);

update products
set stock_qty = stock_qty - 1
where product_id = 206;

update products
set stock_qty = stock_qty - 1
where product_id = 208;

commit;

This ensures that all operations are treated as a single transaction. If any operation fails, the transaction can be rolled back to keep the database consistent.
